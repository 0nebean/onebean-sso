系统权限功能

值得一说的是Kepler中已经预置了一些系统功能也会左右权限的控制，你可以对其稍加利用，便可应对一些常见的业务场景。

禁用功能

Kepler中系统用户和角色都有禁用功能，在你将系统用户禁用以后，该用户将不能登录，并抛出账号不存在的异常。同样你将角色禁用后，已绑定该角色的用户将失去该角色下拥有的权限，并且其他用户将无法绑定该锁定的角色。

    • 用户类型

    Kepler中为用户预置了两种用户类型，一种为面向业务的普通用户角色，另一种为面向开发的系统管理员角色，如果你在用户管理中将某个用户设为管理员类型，那么该用户将拥有所有的菜单权限，这样你就不用频繁的为你新开发的功能添加权限才能进行调试。
