所有模块都依赖spring

Kepler是一个由maven构建的聚合项目，项目整体的父pom又依赖于net.onebean.spring.boot这个基础pom。

net.onebean.spring.boot中对spring boot进行了依赖，并指定了一些全局的版本信息，仅此而已。

和Kepler一样，所有的Kepler的组件也依赖了net.onebean.spring.boot，这样一来所有组件都能以spring风格的api进行编程，也可以简单通过pom依赖低耦合的在项目中进行插拔。

模块含义

Kepler中包含以下子模块

    <modules>
        <module>net.onebean.kepler.dao</module>
        <module>net.onebean.kepler.service</module>
        <module>net.onebean.kepler.security</module>
        <module>net.onebean.kepler.action</module>
        <module>net.onebean.kepler.web</module>
    </modules>

其中各自对应的含义如下，并且他们按照自下而上的顺序依赖。

dao ===>orm层

service ===>服务层 提供了统一的申明式的事务管理

security ===>由spring security实现的权限控制层

action ===>controller层

web ===>web项目层包含了静态资源，模板和项目配置及程序入口
