Kepler是怎么样的框架

Kepler是一个松耦合，多模块的j2ee框架，框架设计思想旨在最做大程度上多端程序共用代码，成为一个高效的企业开发框架。可以说和当下中国流行的RPC通信分布式开发背道而驰。

没错，Kepler本身是一个集中式开发框架，并不能直接作用于分布式系统开发。Kepler代码为求共用，拆分按照实现结构分层分模块，而RPC的模块本身应该是原子性的服务，不应该包含任何冗余功能。

Kepler适应怎么样的业务场景

Kepler技术选型上选择了spring security作为权限控制管理，初衷本是想利用spring security oauth2将Kepler打造成一个兼顾多端程序共用代码和单点登录的技术框架。

这将意味着你的后台管理开发完毕，你的前端项目的API和单点登录几乎已经完成大半。

Kepler更注重效率，市场竞争激烈，很多中小型公司在初始化项目的时候更重注抢占市场和成本，那么凭借该框架一到两名开发就可以在极其短的时间内构建出完整的一套多端产品，试出商业模式好坏。
